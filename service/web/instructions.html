Submission Guide

Runtime Requirements

  • Sampling Limit: Your submission/config.json must set "nsample_per_protein": 1. Submissions with multiple samples per target will be rejected or truncated.
  • Output Format: For every input .fasta in the evaluation bundle, your submission must produce exactly one CIF file at /output/predictions/<target_id>_sampled_0.cif.
  • Local Dev vs. Production: While local development may use MLX for Apple Silicon, MLX is disabled during production benchmark runs. Ensure your train.py defaults to torch or handles backend selection gracefully.
  • Bundle Structure: The uploaded .zip must contain a submission/ folder with at least train.py and config.json. (The repository's baseline_submission/ folder is just a reference; the submitted folder must be named submission/).
Important: Submissions are executed in a network-isolated Docker environment with GPU access (CUDA). Verify your code runs locally using the provided benchmark.sh before uploading.
